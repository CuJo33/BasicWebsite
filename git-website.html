Step 3 - Using the command line

Here is breif description of how to move around files from the command line

In order to know which file location you are currently in you can type

pwd

This stands for Print Working directory, and will return something along the lines of

/c/Users/VR/

You are most likely in the folder of the username that you logged into your computer with ( in this case "VR"). To be able to move through the directories we can use a command called change directory which is a simple command where you supply command with a variable of where you want to go. The command is very simple

cd

cd for Change Directory

Now we want to provide it with a path to follow

cd {FolderName}
cd Desktop

This will now have changed the working directory to be your desktop, again to test this you can use the print working directory command and it should return

/c/Users/VR/Desktop/

When typing these commands you are able to type cd then for instance the first D of Desktop and hit TAB. This will auto complete the word if there is only 1 file or folder with that beginning character. If there are more than 1 result then the program will be unable to complete that task as it can't be sure which file or folder to provide. In this instance you are able to hit TAB twice and then it will return all of the results which fit the parameter as shown below

$ cd D
Desktop/          Documents/        Downloads/

So to be able to navigate through folders quickly you can use the auto complete function if you know your file structures very well

Currently this only tells you how to move down into a folder and not back out of a folder. In order to do this we can pass the change directory command a different parameter

cd ..

By providing the parameter .. we are asking to move out of the current folder that we are in, and move up a level in the hierarchy. It is possible to provide multiple requests to move up many folders at a time, these requests need to be seperated by forward slashes at each folder layer like so

cd ../../..

This will move up the hierarchy 3 levels.This can be seen below by looking at the working directory that is shown after "MINGW64" in the below code after the code has been run to move up and down the folder structure.

VR@VR MINGW64 ~/Desktop/CodingCourse/Work/Project1 (html-code)
    $ cd ../../..
        
VR@VR MINGW64 ~/Desktop
    $ cd CodingCourse/Work/Project1/
        
VR@VR MINGW64 ~/Desktop/CodingCourse/Work/Project1 (html-code)

We are also able to make a new directory (folder) using a command. To make a new directory the command to be used is mkdir followed by the name of the new directory

mkdir {DirectoryName}

You can also create files from the command line

touch {FileName}.{extention}

And remove files

However you must be extremely careful when using remove commands as unless you ask it to the remove command will just do what you say without asking you if you are sure. Therefore you should usually always ask the command to double check it is correct before it runs. you do this by adding "-i" to the command which stands for "interactive" which asks for every deletion to be confirmed.

rm -i {FileName}.{extention}

If you want to learn more about what the command line can do follow this link for a course on it.

Step 4 - Create a repository

I have found it better to do this from the command line rather than within github.

To do this you will need to initiate git using the following command

git init

This will have created a repository within the current folder you are in.

Step 4 - What actually is git?

Next we want to create a file within the folder that we are in.

touch Readme.md

We then want to add this file to git, so that git will try to follow any changes made to it.

git add Readme.MD

Now we want to commit this addition to our repository. We do this using the git commit command, and we want to provide this command a message so that within git version control we get some information on what the changes that are occuring are. This is the "-m" addition to the command.

git commit -m "creation of readme.md"
